Why planar graphs matter for the Four-Color Theorem

The Four-Color Theorem is naturally a theorem about planar maps, but those maps can be translated into planar graphs. In that translation, adjacent regions become adjacent vertices in a suitable graph model, and the coloring problem becomes a graph-coloring problem on planar structures.

Main idea. If every planar graph in the relevant class can be colored with at most four colors, then every planar map can be colored with at most four colors as well.

This is exactly why plantri is interesting here: it gives a fast way to enumerate many isomorphism classes of planar graphs without storing duplicates, so it becomes a useful experimental tool for small cases.

Why brute force does not prove the theorem in full generality

A brute-force strategy would try to check all planar graphs on 1 vertex, then 2 vertices, then 3 vertices, and so on forever. That is the obstacle: the theorem is universal, so a finite check of only small orders can never by itself settle all possible sizes.

Important limitation. Even though complete enumeration is feasible for small numbers of vertices, the Four-Color Theorem concerns an infinite family. Small computations are excellent for building intuition and testing code, but they do not replace the deeper structural arguments used in the actual proof.

Sequential Workflow

Step 1: Download and unpack plantri

This reflects the terminal-style installation sequence you were using

Start in ~/Downloads, fetch the archive, unpack it, and move into the extracted folder.

Download and extract
Matches the flow from your screenshots
cd ~/Downloads
wget https://users.cecs.anu.edu.au/~bdm/plantri/plantri58.tar.gz
tar -xzf plantri58.tar.gz
cd plantri58

If you already extracted the folder earlier, the wget and tar steps do not need to be repeated.

Step 2: Build the program

Compile it once before trying to run ./plantri

The executable must exist before commands like ./plantri ... can work. If you see No such file or directory, it usually means either you are in the wrong folder or the program has not yet been built.

Compile plantri
Usually needed once per extracted source tree
make

A quick check afterward is:

Check that the executable exists
Useful after compilation
ls -l plantri
./plantri -help

Step 3: Read the local guide

You already used this in terminal, which is a good habit

The local guide is worth checking before experimenting with switches.

Open the local guide
Exactly in the style shown in your screenshots
less plantri-guide.txt

One useful thing to remember is that -a chooses ASCII output, which is much easier to inspect by eye than raw planar code.

Step 4: Generate the six-vertex case

This is the small case you were already testing directly in terminal

Once you are inside ~/Downloads/plantri58 and the executable exists, the following command generates connected simple plane graphs on 6 vertices in ASCII form.

Generate 6-vertex connected simple plane graphs
Human-readable ASCII output
./plantri -p -c1m1 -a 6

Here, -p selects general plane simple graphs, -c1m1 asks for connectivity at least 1 and minimum degree at least 1, and -a switches the output to ASCII format.

Step 5: Count first, then list

A very useful habit when the class might be larger than expected

If you want the count without printing the full list first, use -u. Then, once the count looks right, rerun without -u to print the graphs themselves.

Count only
Good first diagnostic step
./plantri -u -p -c1m1 6

Then print the actual graphs:

List the graphs in ASCII form
Readable output after the count check
./plantri -p -c1m1 -a 6

Step 6: Save the output to a file

Convenient if you want to inspect the six-node case later or post-process it in Python

Redirect the ASCII output into a text file in the current folder.

Save ASCII output
Simple text export
./plantri -p -c1m1 -a 6 > six-node-planar-graphs.txt

If you later want a compact abstract graph format instead, you can experiment with graph6 output for simple graphs, but ASCII is the easiest place to start when learning the program.

Why 6 vertices give exactly 179 graphs

In this specific setting, the six-vertex command corresponds to connected simple plane graphs up to embedding-preserving isomorphism. That is exactly the kind of small exhaustive case where plantri is especially useful.

Key small-case fact. For 6 vertices, this class has exactly 179 graphs. That makes it small enough to inspect, draw, color, and place into a PDF atlas, while still being large enough to feel mathematically nontrivial.

This is a good bridge between theory and experiment: large enough to reveal real combinatorial variety, but still small enough to enumerate fully on a laptop.

Practical path mistake to avoid

Common issue from terminal. If you run ./plantri ... from your home directory, or from ~/Downloads before entering the extracted folder, you can get No such file or directory. In your screenshots, the working folder was ~/Downloads/plantri58, which is the correct place once the program has been unpacked and compiled.
Minimal working sequence
From downloads folder to six-node output
cd ~/Downloads
cd plantri58
make
less plantri-guide.txt
./plantri -u -p -c1m1 6
./plantri -p -c1m1 -a 6

Using the generated graphs for Four-Color experiments

Once the graphs are generated, there are several natural follow-up experiments.

  • Draw all 179 graphs in a consistent style.
  • Test each graph with a coloring routine and store one valid 4-coloring.
  • Create a PDF catalogue, for example one graph or a few graphs per page.
  • Compare visual intuition against the theorem: none of these planar cases should require more than four colors.
