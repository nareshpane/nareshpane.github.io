Computational Tests of Four Theorems from Erdős and Renyi (1959)

This page follows the visual format of my earlier graph-theory HTML notes, but it is devoted to Erdős and Renyi’s 1959 paper on random graphs. The paper is short, yet it already contains the threshold picture that later became central to random graph theory: once the edge count enters the critical window, connectivity and related quantities stop behaving vaguely and begin to obey precise limiting laws.

Start with the video. Before looking at formulas, it helps to watch a random graph being built edge by edge. The video below shows a 20-node graph evolving in a partially circular layout with interior nodes, so the eye can follow both the global shape and the small local surprises. As the edge count grows, disconnected fragments begin to fuse, isolated vertices disappear, and the graph moves toward the threshold behavior that Erdős and Renyi made mathematically precise.

The python code for generating the above video is here.

Main idea. Near the edge scale $N_c = \left\lfloor \frac{1}{2} n \log n + cn \right\rfloor$, a random graph moves through a narrow threshold window. Connectivity changes rapidly, the defects near connectivity become countable, and several of those defects converge to Poisson-type laws with mean $e^{-2c}$.

Erdős–Rényi (1959) PDF
Main model
$G(n,m)$ near the connectivity threshold
Computational goal
Empirically validate four asymptotic theorems

Implication of the Paper

The paper shows that random graphs are not just “coin-flip objects.” Near the threshold, there is very rigid structure. Connectivity has a limiting probability curve, the number of vertices outside the giant component becomes asymptotically Poisson, the number of connected components does the same, and even the random time at which connectivity first appears in a dynamic edge-exposure process has a clean asymptotic law.

In practical terms, this means that a computational experiment can be designed theorem by theorem. One does not need to stare at a single graph and guess. One can generate many graphs at carefully chosen edge scales, measure the relevant statistic, and compare the resulting empirical curve or distribution to the paper’s prediction.

Important note. A computer experiment does not formally prove an asymptotic theorem. What it does is give controlled numerical evidence in the same regime as the theorem, so one can see how quickly the limiting formulas begin to emerge for finite values such as $n=50,100,200,500$.

Theorem 1: Connectivity at the Threshold

What the theorem is about

The probability of being connected converges to a specific curve

Theorem 1 studies the event that a graph in $G(n,N_c)$ is connected. The paper shows

$$P_0(n,N_c) \to e^{-e^{-2c}}.$$

So $c$ tells us where we are in the connectivity threshold window. Negative $c$ means we are still on the sparse side, while positive $c$ moves us into the connected regime.

What it means to test this computationally

Estimate a probability curve rather than inspect one graph

For each $(n,c)$ pair, sample many graphs with exactly $m=N_c$ edges, measure the fraction that are connected, and compare that fraction to the theorem curve $e^{-e^{-2c}}$.

My computational strategy

Direct Monte Carlo in the fixed-edge model
  • Work in $G(n,m)$ rather than $G(n,p)$.
  • Set $m=N_c=\left\lfloor \frac{1}{2}n\log n + cn \right\rfloor$.
  • Estimate connectivity probability for a grid of $c$ values.
  • Overlay empirical curves against the theorem and record absolute differences.
Minimalist reading of the threshold picture. When $c$ is negative, the graph is usually still missing enough links that isolated vertices or tiny fragments survive. As $c$ moves through the narrow critical window, the connectivity probability rises sharply rather than gradually. Once $c$ becomes positive, those last defects are rapidly washed away, which is exactly why the limit curve $e^{-e^{-2c}}$ becomes the natural comparison target.
Python code for Theorem 1
Connectivity in $G(n,m)$ near the Erdős–Renyi threshold
from __future__ import annotations

import argparse
import math
from dataclasses import dataclass
from pathlib import Path
from typing import Iterable

import matplotlib.pyplot as plt
import networkx as nx
import numpy as np


@dataclass(frozen=True)
class ResultRow:
    n: int
    c: float
    m: int
    theory: float
    empirical: float
    abs_diff: float
    trials: int


def edge_count_n_c(n: int, c: float) -> int:
    """Return N_c = floor(0.5 * n * log n + c n), clipped to [0, n choose 2]."""
    m = math.floor(0.5 * n * math.log(n) + c * n)
    max_edges = n * (n - 1) // 2
    return max(0, min(m, max_edges))



def theorem1_limit(c: float) -> float:
    """Theorem 1 limit for connectivity in G(n, N_c)."""
    return math.exp(-math.exp(-2.0 * c))



def estimate_connectivity_probability(
    n: int,
    m: int,
    trials: int,
    rng: np.random.Generator,
) -> float:
    """Monte Carlo estimate of P[G(n,m) is connected]."""
    connected = 0
    for _ in range(trials):
        seed = int(rng.integers(0, 2**32 - 1, dtype=np.uint32))
        g = nx.gnm_random_graph(n, m, seed=seed)
        if nx.is_connected(g):
            connected += 1
    return connected / trials



def run_experiment(
    n_values: Iterable[int],
    c_values: Iterable[float],
    trials: int,
    seed: int,
) -> list[ResultRow]:
    rng = np.random.default_rng(seed)
    rows: list[ResultRow] = []
    for n in n_values:
        for c in c_values:
            c_float = float(c)
            m = edge_count_n_c(n, c_float)
            empirical = estimate_connectivity_probability(n, m, trials, rng)
            theory = theorem1_limit(c_float)
            abs_diff = abs(theory - empirical)
            rows.append(
                ResultRow(
                    n=n,
                    c=c_float,
                    m=m,
                    theory=theory,
                    empirical=empirical,
                    abs_diff=abs_diff,
                    trials=trials,
                )
            )
    return rows



def save_csv(rows: list[ResultRow], path: Path) -> None:
    with path.open("w", encoding="utf-8") as f:
        f.write("n,c,m,theory,empirical,abs_diff,trials\n")
        for row in rows:
            f.write(
                f"{row.n},{row.c:.6f},{row.m},{row.theory:.10f},{row.empirical:.10f},{row.abs_diff:.10f},{row.trials}\n"
            )



def plot_results(rows: list[ResultRow], output_path: Path) -> None:
    plt.figure(figsize=(10.5, 6.2))

    c_curve = np.linspace(min(r.c for r in rows), max(r.c for r in rows), 400)
    theory_curve = np.exp(-np.exp(-2.0 * c_curve))
    plt.plot(
        c_curve,
        theory_curve,
        linewidth=2.5,
        label=r"Theorem 1 limit $e^{-e^{-2c}}$",
    )

    markers = {50: "o", 100: "s", 200: "^", 500: "D"}
    for n in sorted({r.n for r in rows}):
        subset = sorted((r for r in rows if r.n == n), key=lambda r: r.c)
        x = [r.c for r in subset]
        y = [r.empirical for r in subset]
        labels = [str(r.m) for r in subset]
        plt.plot(x, y, marker=markers.get(n, "o"), linewidth=1.4, label=fr"Empirical, $n={n}$")

        mid = min(range(len(subset)), key=lambda i: abs(subset[i].c))
        plt.annotate(
            fr"$m={labels[mid]}$",
            (x[mid], y[mid]),
            textcoords="offset points",
            xytext=(8, 8),
            fontsize=8,
        )

    plt.xlabel(r"$c$ in $N_c = \lfloor \frac{1}{2} n \log n + cn \rfloor$")
    plt.ylabel("Connectivity probability")
    plt.title("Erdős–Rényi (1959), Theorem 1: connectivity in $G(n,m)$")
    plt.ylim(-0.02, 1.02)
    plt.grid(alpha=0.3)
    plt.legend()
    plt.tight_layout()
    plt.savefig(output_path, dpi=200)
    plt.close()



def print_summary(rows: list[ResultRow]) -> None:
    print("n      c        m       theory        empirical      abs_diff")
    print("-" * 72)
    for row in rows:
        print(
            f"{row.n:<5d} {row.c:>6.2f}   {row.m:<5d}   {row.theory:>10.6f}   {row.empirical:>10.6f}   {row.abs_diff:>10.6f}"
        )

Theorem 2: Vertices Outside the Largest Connected Component

What the theorem is about

The deficit outside the giant component becomes Poisson

Let $P_k(n,N_c)$ denote the probability that the largest connected component has exactly $n-k$ vertices. The theorem states

$$P_k(n,N_c) \to \frac{(e^{-2c})^k e^{-e^{-2c}}}{k!}.$$

In plain language, the number of vertices left outside the largest connected component behaves asymptotically like a Poisson random variable with mean $e^{-2c}$.

What it means to test this computationally

We now test a whole distribution rather than one event

The measured quantity is $$k = n - |C_{\max}|,$$ where $|C_{\max}|$ is the largest component size. Since the display window cannot be infinite, the code records $k=0,1,\dots,10$ explicitly and stores the remaining probability mass as a tail for $k>10$.

My computational strategy

Track the largest component directly and keep the omitted tail visible
  • Generate random graphs at the threshold edge count $N_c$.
  • Compute the largest connected component for each graph.
  • Convert that to $k=n-|C_{\max}|$.
  • Estimate probabilities for $k=0$ through $10$ and record the tail probability beyond that range.

Distribution of the number of vertices outside the largest connected component
from __future__ import annotations

import argparse
import math
from dataclasses import dataclass
from pathlib import Path
from typing import Iterable

import matplotlib.pyplot as plt
import networkx as nx
import numpy as np


@dataclass(frozen=True)
class ResultRow:
    n: int
    c: float
    m: int
    k: int
    theory: float
    empirical: float
    abs_diff: float
    trials: int


@dataclass(frozen=True)
class TailRow:
    n: int
    c: float
    m: int
    tail_prob: float
    trials: int


def edge_count_n_c(n: int, c: float) -> int:
    """Return N_c = floor(0.5 * n * log n + c n), clipped to [0, n choose 2]."""
    m = math.floor(0.5 * n * math.log(n) + c * n)
    max_edges = n * (n - 1) // 2
    return max(0, min(m, max_edges))



def theorem2_limit(c: float, k: int) -> float:
    """Theorem 2 limit for P_k(n, N_c): Poisson(e^{-2c}) at k."""
    lam = math.exp(-2.0 * c)
    return math.exp(-lam) * (lam**k) / math.factorial(k)



def estimate_deficit_distribution(
    n: int,
    m: int,
    k_max: int,
    trials: int,
    rng: np.random.Generator,
) -> tuple[np.ndarray, float]:
    """
    Estimate probabilities for k = 0, ..., k_max where
    k = n - size_of_largest_connected_component.

    Returns (probs, tail_prob), where probs has length k_max + 1 and
    tail_prob is the fraction of trials with k > k_max.
    """
    counts = np.zeros(k_max + 1, dtype=np.int64)
    tail = 0

    for _ in range(trials):
        seed = int(rng.integers(0, 2**32 - 1, dtype=np.uint32))
        g = nx.gnm_random_graph(n, m, seed=seed)
        largest_component = max((len(comp) for comp in nx.connected_components(g)), default=0)
        deficit = n - largest_component
        if 0 <= deficit <= k_max:
            counts[deficit] += 1
        else:
            tail += 1

    probs = counts / trials
    tail_prob = tail / trials
    return probs, tail_prob



def run_experiment(
    n_values: Iterable[int],
    c_values: Iterable[float],
    k_max: int,
    trials: int,
    seed: int,
) -> tuple[list[ResultRow], list[TailRow]]:
    rng = np.random.default_rng(seed)
    rows: list[ResultRow] = []
    tail_rows: list[TailRow] = []

    for n in n_values:
        for c in c_values:
            c_float = float(c)
            m = edge_count_n_c(n, c_float)
            empirical_probs, tail_prob = estimate_deficit_distribution(n, m, k_max, trials, rng)
            tail_rows.append(TailRow(n=n, c=c_float, m=m, tail_prob=tail_prob, trials=trials))

            for k in range(k_max + 1):
                theory = theorem2_limit(c_float, k)
                empirical = float(empirical_probs[k])
                abs_diff = abs(theory - empirical)
                rows.append(
                    ResultRow(
                        n=n,
                        c=c_float,
                        m=m,
                        k=k,
                        theory=theory,
                        empirical=empirical,
                        abs_diff=abs_diff,
                        trials=trials,
                    )
                )

    return rows, tail_rows

def plot_results_by_k(rows: list[ResultRow], output_dir: Path) -> None:

    c_min = min(r.c for r in rows)
    c_max = max(r.c for r in rows)

    for k in k_values:

        theory_curve = np.array([theorem2_limit(float(c), k) for c in c_curve])

            theory_curve,
            linewidth=2.5,
            label=fr"Theorem 2 limit for $k={k}$",
        )

        for n in sorted({r.n for r in rows}):

        )
        current_group = group

            f"{'':<5} {'':>6}   {'':<5}   >{k_max:<2d}   {'':>10}   {tail.tail_prob:>10.6f}   {'':>10}"
        )

Theorem 3: Number of Connected Components Minus One

What the theorem is about

A second Poisson law, now for the total component count

Let $\Pi_k(n,N_c)$ denote the probability that the graph has exactly $k+1$ connected components. Then

$$\Pi_k(n,N_c) \to \frac{(e^{-2c})^k e^{-e^{-2c}}}{k!}.$$

So the number of connected components minus one also converges to a Poisson law with mean $e^{-2c}$.

What it means to test this computationally

Use the same threshold window, but measure a different statistic

Now the variable is $$k = \#\text{components} - 1.$$ This looks close to Theorem 2, but it is not the same quantity and deserves its own experiment and plots.

My computational strategy

Count components directly and compare to the theorem’s Poisson limit
  • Keep the same $(n,c)$ grid as in Theorem 2.
  • Count the exact number of connected components in each sampled graph.
  • Subtract one and tabulate $k=0,1,\dots,10$ plus a tail.

def estimate_component_count_distribution(
    n: int,
    m: int,
    k_max: int,
    trials: int,
    rng: np.random.Generator,

    tail_prob = tail / trials
    return probs, tail_prob

Theorem 4: The Edge-Exposure Process and the Hitting Time of Connectivity

What the theorem is about

A dynamic process replaces the static graph model

Here the edges of $K_n$ are exposed one at a time, each next unused edge chosen uniformly at random, and the process stops the moment the graph first becomes connected. If $\nu_n$ is that stopping time in edge count, the paper gives both a local asymptotic and a limiting CDF:

$$P\!\left(\nu_n = \left\lfloor \frac{1}{2}n\log n \right\rfloor + l\right) \sim \frac{2}{n} e^{-2l/n - e^{-2l/n}},$$ $$P\!\left(\frac{\nu_n - \frac{1}{2}n\log n}{n} < x\right) \to e^{-e^{-2x}}.$$

What it means to test this computationally

We must simulate the stopping-time process itself

This theorem is the most delicate one computationally. It is no longer enough to sample from $G(n,m)$. The code has to reveal edges one by one and stop exactly when the graph first becomes connected.

My computational strategy

Test both the CDF statement and the local point-mass statement
  • Simulate the random edge-exposure process directly.
  • Use a union-find structure to monitor connectivity efficiently.
  • Record $\nu_n$ for many trials.
  • Compare the empirical CDF to $e^{-e^{-2x}}$.
  • Compare the local probabilities to the asymptotic point-mass formula, including a scaled comparison that behaves better across different $n$.

# ---------------------------------------------------------
# Erdős–Rényi (1959), Theorem 4
#
# Random process:
# Choose edges of K_n successively, each next edge uniformly among the
# edges not yet chosen, until the graph becomes connected.
#
# Let ν_n be the number of chosen edges when connectivity first appears.
#
# The theorem states, in the notation of the paper:
#   P(ν_n = floor(0.5*n*log n) + l) ~ (2/n) * exp(-2l/n - exp(-2l/n))
# for |l| = O(n),
# and equivalently
#   P((ν_n - 0.5*n*log n)/n < x) -> exp(-exp(-2x)).
#
# This script tests both:
#   1) the CDF statement on an x-grid;
#   2) the local point-mass statement on an l-grid induced by x-grid.
#
# The structure intentionally mirrors the earlier theorem scripts.
# ---------------------------------------------------------

class UnionFind:
    __slots__ = ("parent", "size", "components")

    def __init__(self, n: int) -> None:
        self.parent = list(range(n))
        self.size = [1] * n
        self.components = n

    def find(self, x: int) -> int:
        parent = self.parent
        while parent[x] != x:
            parent[x] = parent[parent[x]]
            x = parent[x]
        return x

    def union(self, a: int, b: int) -> bool:
        ra = self.find(a)
        rb = self.find(b)
        if ra == rb:
            return False
        if self.size[ra] < self.size[rb]:
            ra, rb = rb, ra
        self.parent[rb] = ra
        self.size[ra] += self.size[rb]
        self.components -= 1
        return True


def theorem4_cdf_limit(x: float) -> float:
    """Limit CDF in Theorem 4: exp(-exp(-2x))."""
    return math.exp(-math.exp(-2.0 * x))


def theorem4_local_limit_raw(n: int, l: int) -> float:
    """
    Local mass approximation in Theorem 4:
    P(ν_n = floor(0.5*n*log n) + l) ~ (2/n) exp(-2l/n - exp(-2l/n)).
    """
    x = l / n
    return (2.0 / n) * math.exp(-2.0 * x - math.exp(-2.0 * x))


def theorem4_local_limit_scaled(x: float) -> float:
    """
    Scaled local mass:
    (n/2) P(ν_n = floor(0.5*n*log n) + l) ~ exp(-2x - exp(-2x)),
    where x = l/n.
    """
    return math.exp(-2.0 * x - math.exp(-2.0 * x))


def build_edge_array(n: int) -> np.ndarray:
    """Return all edges of K_n as an array of shape (binom(n,2), 2)."""
    return np.array(list(combinations(range(n), 2)), dtype=np.int32)


def simulate_connectivity_hitting_time(
    n: int,
    edges: np.ndarray,
    rng: np.random.Generator,
) -> int:
    """
    Simulate ν_n:
    reveal distinct edges uniformly without replacement until connected.
    We implement this by rejection sampling edge indices; the accepted next
    edge is uniform among the remaining unchosen edges.
    """
    total_edges = len(edges)
    seen: set[int] = set()
    uf = UnionFind(n)
    chosen = 0

    while uf.components > 1:
        eid = int(rng.integers(0, total_edges))
        if eid in seen:
            continue
        seen.add(eid)
        u, v = edges[eid]
        uf.union(int(u), int(v))
        chosen += 1

    return chosen


def sample_hitting_times(
    n: int,
    trials: int,
    rng: np.random.Generator,
) -> np.ndarray:
    """Return an array of ν_n samples of length trials."""
    edges = build_edge_array(n)
    vals = np.empty(trials, dtype=np.int32)
    for t in range(trials):
        vals[t] = simulate_connectivity_hitting_time(n, edges, rng)
    return vals


def build_cdf_rows(
    n: int,
    samples: np.ndarray,
    x_values: Iterable[float],
    trials: int,
) -> list[CDFRow]:
    rows: list[CDFRow] = []
    center = 0.5 * n * math.log(n)

    for x in x_values:
        threshold = math.floor(center + x * n)
        empirical = float(np.mean(samples <= threshold))
        theory = theorem4_cdf_limit(float(x))
        rows.append(
            CDFRow(
                n=n,
                x=float(x),
                threshold=threshold,
                theory=theory,
                empirical=empirical,
                abs_diff=abs(theory - empirical),
                trials=trials,
            )
        )
    return rows


def build_local_rows(
    n: int,
    samples: np.ndarray,
    x_values: Iterable[float],
    trials: int,
) -> list[LocalRow]:
    rows: list[LocalRow] = []
    base = math.floor(0.5 * n * math.log(n))

    for x in x_values:
        l = int(round(float(x) * n))
        m = base + l
        empirical = float(np.mean(samples == m))
        theory = theorem4_local_limit_raw(n, l)
        scaled_theory = theorem4_local_limit_scaled(l / n)
        scaled_empirical = 0.5 * n * empirical
        rows.append(
            LocalRow(
                n=n,
                x=l / n,
                l=l,
                m=m,
                theory=theory,
                empirical=empirical,
                abs_diff=abs(theory - empirical),
                scaled_theory=scaled_theory,
                scaled_empirical=scaled_empirical,
                scaled_abs_diff=abs(scaled_theory - scaled_empirical),
                trials=trials,
            )
        )
    return rows

Final Perspective

The four theorems fit together naturally. Theorem 1 describes the connectivity threshold itself. Theorem 2 explains how many vertices remain outside the giant component near that threshold. Theorem 3 translates the same threshold picture into a statement about the number of connected components. Theorem 4 shifts to a dynamic exposure process and shows that the same threshold structure still governs the first moment of connectivity.

That is what makes this paper so attractive computationally: it is rigorous, asymptotic, and conceptually elegant, but it is also concrete enough that one can write transparent code and watch the limiting laws begin to emerge numerically.
