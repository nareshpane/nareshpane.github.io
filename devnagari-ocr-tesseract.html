Conversion of Devnagari Language from scanned PDF into Editable Text or Word Document Using Tesseract

This page walks through a Linux-first workflow for converting scanned Devnagari or Hindi PDF pages into editable text, and then into a Word document. The goal here is practical: install only what is needed, check that the Hindi OCR data is actually available, test on one image first, and then run the full PDF pipeline cleanly.

What this page is trying to do. A lot of OCR guides are too vague. Here the point is more direct: use the commands that actually worked in terminal, trim redundant steps, and give a sequence that another Linux user can copy without guessing too much.

Sequential Workflow

Step 1: Install Tesseract and the Hindi OCR Language Pack

Start with the OCR engine first, and make sure Hindi support is included

The first thing is to have Tesseract itself installed, along with the Hindi language pack. This is the part that matters most. Without the Hindi OCR data, the command may run, but it will not properly recognize Devnagari text.

On Ubuntu, this is the clean starting point:

Install Tesseract for Hindi / Devnagari OCR
Core packages only
sudo apt update
sudo apt install tesseract-ocr
sudo apt install tesseract-ocr-hin

If a user wants a single combined install line later, that is fine too, but splitting it like this makes it easier to see whether the Hindi package is the missing piece.

Step 2: Run Basic Checks Before Doing Any OCR

Confirm that Tesseract is installed and that Hindi appears in the language list

Before touching the PDF, do two quick checks. First, confirm the version. Second, list the available OCR languages. This is often where Linux configuration issues show up.

Version and language checks
Useful when troubleshooting setup issues
tesseract --version
tesseract --list-langs

What you want to see is that Tesseract is installed, and that hin appears in the language list. If hin is missing, install tesseract-ocr-hin again and check once more.

Step 3: Do a Quick One-Image OCR Test First

This is a good sanity check before working on the full PDF

A quick test on one PNG image is worth doing. It tells you right away whether Hindi OCR is basically functioning on your machine. This is especially helpful before you bring in PDF conversion and batch processing.

Single-image Hindi OCR test
Replace the path with your own image path
tesseract /home/naresh/Pictures/Screenshots/bhupi1.png output -l hin

That creates a file named output.txt in the current directory. If that works, then the OCR engine and Hindi language data are basically in place.

Step 4: Install the Extra PDF and DOCX Utilities

Use poppler for PDF page extraction and pandoc for DOCX conversion

Once the OCR engine itself is working, the next useful tools are:

  • poppler-utils for turning PDF pages into PNG images,
  • pandoc for converting the final text output into a Word document.
Install supporting tools
For PDF page extraction and DOCX output
sudo apt install tesseract-ocr tesseract-ocr-hin poppler-utils pandoc

This line is slightly redundant if the first packages are already installed, but it is still a clean “all required tools” command for other readers.

Step 5: Move Into the Working Directory and Extract the PDF into PNG Pages

This is the cleanest way to handle a scanned PDF before OCR

Once the PDF is ready, go to the folder where you want the output to live, create a pages folder, and convert the scanned PDF into one PNG file per page.

Create page images from the scanned PDF
Replace the PDF filename if yours differs
cd /home/naresh/Documents
mkdir -p bhupi_pages
pdftoppm -png BhupiSherchan_GhumneMechMathi.pdf bhupi_pages/page

After that, the folder bhupi_pages should contain PNG files such as page-1.png, page-2.png, and so on.

Step 6: Run OCR Across All Extracted Pages

Process each page image and append the recognized text into one output file

This is the main batch OCR step. It loops through each page image, runs Hindi OCR, and appends the recognized text into a single text file, with a visible page break marker in between.

Batch OCR loop for all pages
This is the key working block
cd /home/naresh/Documents
rm -f bhupi_ocr.txt

for f in bhupi_pages/*.png; do
    echo "Processing: $f"
    tesseract "$f" stdout -l hin --psm 6 >> bhupi_ocr.txt
    printf "\n\n==== PAGE BREAK ====\n\n" >> bhupi_ocr.txt
done

The use of stdout is helpful here because it sends the OCR text directly into the combined output file. The option --psm 6 is a useful starting page segmentation mode for scanned text blocks.

Step 7: Convert the OCR Text File into a Word Document

Once the text file exists, turn it into a DOCX file

After the OCR text is collected in one file, the next step is straightforward. Use pandoc to convert the text file into a Word document.

TXT to DOCX conversion
Simple and direct final export
pandoc bhupi_ocr.txt -o BhupiSherchan_GhumneMechmathi.docx

At that point, you have both a plain text OCR output and a Word document version that can be edited further.

Step 8: Extra Checks if the OCR Looks Strange

Useful Linux-side checks when the setup works but the text output is poor

A few extra checks can save time when the command runs but the output is still poor:

  • Confirm again that hin is in tesseract --list-langs.
  • Make sure the PDF really is scanned-image based, not already text-based.
  • Inspect one extracted PNG page manually before processing the whole document.
  • If pages are tiny, blurred, cropped, or skewed, OCR quality can drop sharply.
Quick re-check commands
Handy when another user’s Linux setup behaves differently
tesseract --version
tesseract --list-langs
ls bhupi_pages
head -n 40 bhupi_ocr.txt

Common Pitfall

If you see errors like “Image too small to scale!!”, that usually points to a bad or unexpected input image rather than a Hindi language-pack issue by itself. It can happen if a page image is tiny, corrupted, empty, or not what the loop expects. In that case, check the extracted PNG pages directly before re-running the OCR loop.

Minimal End-to-End Command Set

For readers who want the whole workflow in one place, this is the compact version:

Compact workflow
Install, extract, OCR, convert
sudo apt update
sudo apt install tesseract-ocr tesseract-ocr-hin poppler-utils pandoc

tesseract --version
tesseract --list-langs

cd /home/naresh/Documents
mkdir -p bhupi_pages
pdftoppm -png BhupiSherchan_GhumneMechMathi.pdf bhupi_pages/page

rm -f bhupi_ocr.txt
for f in bhupi_pages/*.png; do

    tesseract "$f" stdout -l hin --psm 6 >> bhupi_ocr.txt
    printf "\n\n==== PAGE BREAK ====\n\n" >> bhupi_ocr.txt
done

pandoc bhupi_ocr.txt -o BhupiSherchan_GhumneMechmathi.docx
